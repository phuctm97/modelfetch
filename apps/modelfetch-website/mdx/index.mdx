ModelFetch makes it effortless to deploy your MCP servers anywhere. Just pass your existing [`McpServer`](https://github.com/modelcontextprotocol/typescript-sdk?tab=readme-ov-file#server) instance to ModelFetch's `handle()` function, and it takes care of all the runtime-specific details. No need to learn new APIs or rewrite your codeâ€”if you've built an MCP server with the [official MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk), you're already ready to deploy with ModelFetch.

## How It Works

ModelFetch works with any [`McpServer`](https://github.com/modelcontextprotocol/typescript-sdk?tab=readme-ov-file#server) instance from the [official MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk). Here's all it takes:

### 1. Create your MCP server

```typescript title="server.ts"
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { z } from "zod";

const server = new McpServer({
  title: "My MCP Server",
  name: "my-server",
  version: "1.0.0",
});

server.registerTool(
  "roll_dice",
  {
    title: "Roll Dice",
    description: "Rolls an N-sided dice",
    inputSchema: { sides: z.number().int().min(2) },
  },
  ({ sides }) => ({
    content: [
      {
        type: "text",
        text: `ðŸŽ² You rolled a ${1 + Math.floor(Math.random() * sides)}!`,
      },
    ],
  }),
);

export default server;
```

### 2. Add ModelFetch to deploy it anywhere

```typescript title="index.ts"
import handle from "@modelfetch/node"; // Choose your runtime
import server from "./server";

// That's it! ModelFetch handles all runtime-specific details
handle(server);
```

Just two lines of code, and your MCP server runs on any supported platform. No need to learn platform-specific APIs or rewrite your server code.

## Available Packages

ModelFetch provides runtime-specific packages for deploying MCP servers:

- `@modelfetch/node` - Node.js runtime support
- `@modelfetch/bun` - Bun runtime support
- `@modelfetch/deno` - Deno runtime support
- `@modelfetch/cloudflare` - Cloudflare Workers support
- `@modelfetch/vercel` - Vercel Functions support
